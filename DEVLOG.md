## 架构说明

**当前**：单文件 HTML PWA（`coffee_v6.html`）
- 所有 CSS / HTML / JS 内联
- 数据层：`localStorage`（key: `cf6`）
- AI 层：直接 fetch `api.anthropic.com`，无后端

**GitHub 语言占比问题**：当前 98% HTML 是单文件架构的必然结果。
策略：在 v6.x 阶段专注功能迭代，完整验证后在 v7 重构时拆分：
- `style.css` ← 抽出所有样式
- `app.js` ← 抽出所有 JS 逻辑
- `index.html` ← 只保留骨架 HTML

重构后 HTML 占比预计降至 15% 以下。

## v6.1-star · 风味星图（计划中）

### 思考

用户反馈当前"气泡标签"不直观、不吸引人。
Gemini 建议做 3D 坐标系（Three.js），经讨论认为：
- 移动端 3D 散点图认知负担重，用户难以直接读出意义
- 更好的隐喻：**星图**——每杯咖啡是一颗星，越喝越多，星点越密，自然形成属于用户的星座轮廓

### 核心理念

> 不是在积累数据，是在帮用户看见自己的味觉宇宙

### 方向

- 用 Canvas API（原生，无依赖，支持发光/模糊特效）
- 两轴感官定位，从 feel 文本关键词提取（无需额外 API 调用）
  - X 轴：酸质（明亮柑橘）←→ 醇苦（巧克力烟熏）
  - Y 轴：轻盈（花香茶感）←→ 浓郁（厚重油脂）
- 每颗星属性：位置=感官坐标 / 大小=评分+wow / 颜色=烘焙度 / wow=金色发光
- 连线：按时间顺序淡淡相连，形成轨迹感
- 最密集区：柔光晕染 = 用户的"味觉家园"

### 视觉风格

深咖色背景（`#0B0806`）+ 暖琥珀星点 + 颗粒纹理
颜色体系：浅烘→琥珀金 / 深烘→奶油深棕 / wow时刻→金色光晕

### 实现方式

1. 在 `bubble-card` 位置插入 `<canvas id="star-map">` + 交互提示
2. `renderStarMap()` 函数：读取所有带 feel 的记录 → 提取感官坐标 → Canvas 绘制
3. `extractSensoryVector(feel)` 关键词权重函数（本地，零 API）
4. 保留气泡词云作为星图下方辅助信息

### 备份策略

推送前 git commit 当前状态，iCloud 保留 `coffee_v6.html` 原始副本。

## v6.4-star · 星图交互重构（2026-04-04 完成）

### 问题

星点点击后信息浮层叠在 Canvas 上：背景是暗色星图，文字对比度差，内容一多就挤压可读空间，位置还会跑到边缘被裁。

**根本原因**：浮层和画布争抢同一块视觉空间，注定是零和博弈。

### 方向决策

不在星图上叠任何文字内容，**画布只做画布**。把所有叙述性信息移到星图卡片的下方区域——点击星点后底部抽屉滑入，星图始终完整可见。

同时废弃独立的"店铺足迹"列表：用户要的不是一张清单，而是理解"我在哪里喝到了什么感受"。店铺名进入抽屉，作为每条记录的锚点，而不是独立聚合成一份访店报告。

### 产品理念：记忆是有生命的

抽屉不只是展示，而是**可编辑的回忆入口**。

用户的口味认知会随时间成长。第一次喝耶加可能只写"有点酸，还不错"，半年后回看，那其实是柑橘调的日晒、轻盈干净。允许回头修改感受描述，AI 重新加工，这条记录就从静态快照变成了有生命的记忆。

这和"修改记录"是不同的心智模型——不是纠错，而是**用现在更成熟的语言，重新诠释过去的那一杯**。

### AI 体会一句话

不照搬用户原话，而是"帮他说出口"：提炼感受核心，用老朋友的语气表达，而非专业点评。

实现策略：
- 新记录保存时后台生成（不阻塞主流程），存入 `rec.summary`
- 老记录首次点击时懒加载，生成后持久化到 localStorage
- 无网络/无 API Key 时降级显示原始 feel 文本

### 顺带修复

`getReply` 和 `typewrite` 在某次重构中被意外删除，导致保存记录后 AI 回复功能失效。本次一并恢复。
